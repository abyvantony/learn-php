# Level 5: Arrays: indexed, associative, and the kitchen sink

**Track:** PHP · **Level:** 5/12 · **Difficulty:** `beginner-plus`

📚 **Today's lesson** — published 2026-06-28

## TL;DR

PHP arrays are actually ordered maps — they work as both a list (numeric keys) and a dictionary (string keys). This dual nature is PHP's superpower and its curse.

## Real-world analogies

- An indexed array is a numbered list (1, 2, 3...). An associative array is a labeled filing cabinet where each drawer has a name.
- PHP's array is a Swiss Army knife — list, map, stack, queue, set all rolled into one type.

## Key concepts

### ``array()` vs `[]``

Both create arrays. The square bracket syntax is shorter and modern. `[]` is also used for short array syntax in PHP 5.4+.

### `Mixed keys`

You CAN mix numeric and string keys in one array. You probably shouldn't — it's confusing.

### ``array_push` vs `$arr[] =``

`$arr[] = $value` is faster than `array_push($arr, $value)`. Use the bracket syntax.

### `Spread operator (PHP 7.4+)`

`[...$a, ...$b]` merges arrays. Cleaner than `array_merge` for simple cases.

## Code with comments

Every line has a comment. Read it slowly.

```php
<?php
// Indexed array (numeric keys 0, 1, 2...)
$colors = ["red", "green", "blue"];
echo $colors[0];  // "red"

// Associative array (string keys)
$user = [
    "name" => "Aby",
    "age" => 30,
    "city" => "Trivandrum"
];
echo $user["name"];  // "Aby"

// Mixed (legal but messy)
$mixed = [0 => "a", "name" => "b", 1 => "c"];

// Useful array functions
$fruits = ["apple", "banana", "cherry"];

count($fruits);                // 3
in_array("banana", $fruits);   // true
array_search("cherry", $fruits); // 2 (the index)
sort($fruits);                 // sort in place, now ["apple", "banana", "cherry"]
rsort($fruits);                // reverse sort
shuffle($fruits);              // randomize

// Associative-specific
$user = ["name" => "Aby", "age" => 30];
array_keys($user);             // ["name", "age"]
array_values($user);           // ["Aby", 30]
array_key_exists("name", $user);  // true
isset($user["name"]);          // true (also true for null — careful!)

// Adding/removing
$arr = [1, 2, 3];
$arr[] = 4;                    // [1, 2, 3, 4]
array_push($arr, 5);           // [1, 2, 3, 4, 5]
array_pop($arr);               // removes last, returns it → [1, 2, 3, 4]
array_shift($arr);             // removes first → [2, 3, 4]
array_unshift($arr, 0);        // adds to front → [0, 2, 3, 4]
unset($arr[1]);                // removes by key → [0 => 0, 2 => 3, 3 => 4]
                               // NOTE: leaves a "hole" in numeric keys

// Spread operator
$a = [1, 2, 3];
$b = [4, 5, 6];
$merged = [...$a, ...$b];      // [1, 2, 3, 4, 5, 6]

// Destructuring (PHP 7.1+)
[$first, $second, $third] = $fruits;

// Array destructuring with associative (PHP 7.1+)
["name" => $name, "age" => $age] = $user;
echo "$name is $age";  // Aby is 30
```

## Try it yourself

Build a contacts list: an array of associative arrays, each with name, phone, and email. Write a function that searches the list by name and returns the matching contact. Use `array_filter` or a loop.

## Common pitfalls

- ⚠️ `array_search` returns `false` on miss and `0` (falsy) on hit-at-index-0. Use `!== false` to check, never `!=`.
- ⚠️ `isset($arr[$key])` is true even if the value is null. Use `array_key_exists` if you need to distinguish.
- ⚠️ Sorting functions return true/false but operate in-place — they don't return a new array. `$sorted = sort($arr);` makes `$sorted = true` and `$arr` is sorted.
