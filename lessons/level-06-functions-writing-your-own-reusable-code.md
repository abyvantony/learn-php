# Level 6: Functions: writing your own reusable code

**Track:** PHP · **Level:** 6/12 · **Difficulty:** `beginner-plus`

📚 **Today's lesson** — published 2026-06-29

## TL;DR

Functions package a chunk of code so you can call it by name. They take inputs (parameters) and can return one value. They make code reusable, testable, and readable.

## Real-world analogies

- A function is a recipe. You write it once (the function definition), and every time you 'make dinner' (call the function), you follow the same steps with different ingredients (arguments).
- Parameters are the recipe's blank lines: 'add ___ cups of flour'. Arguments are the actual numbers you fill in when you cook.

## Key concepts

### `Return vs echo`

`echo` prints to the browser immediately. `return` hands a value back to whoever called the function. Prefer `return` — let the caller decide what to do with the result.

### `Default parameters`

`function greet($name = 'World')` makes `$name` optional. If the caller doesn't pass it, the default is used.

### `Type declarations`

PHP 7+ lets you declare types: `function add(int $a, int $b): int`. PHP will throw an error if wrong types are passed (with strict_types enabled).

### `Variable scope`

Variables defined inside a function are local — they don't exist outside. Use `global $x` (avoid) or pass values in/out to share state.

## Code with comments

Every line has a comment. Read it slowly.

```php
<?php
// Basic function
function greet($name) {
    return "Hello, $name!";
}
echo greet("Aby");  // "Hello, Aby!"

// Default parameter
function greetWithTime($name, $time = "morning") {
    return "Good $time, $name!";
}
echo greetWithTime("Aby");              // "Good morning, Aby!"
echo greetWithTime("Aby", "evening");   // "Good evening, Aby!"

// Type declarations (PHP 7+)
function add(int $a, int $b): int {
    return $a + $b;
}
add(2, 3);        // 5
add("2", "3");    // 5 (PHP coerces strings to int) — works without strict_types
                 // With declare(strict_types=1), this throws TypeError

// Nullable types (PHP 7.1+)
function findUser(int $id): ?User {
    return $id === 1 ? new User() : null;
}
// ?User means the return is User OR null

// Union types (PHP 8+)
function process(int|string $input): int|false {
    return is_numeric($input) ? (int)$input : false;
}

// Variadic functions (PHP 5.6+)
function sum(int ...$numbers): int {
    $total = 0;
    foreach ($numbers as $n) {
        $total += $n;
    }
    return $total;
}
sum(1, 2, 3, 4, 5);  // 15
sum(10, 20);          // 30

// Arrow functions (PHP 7.4+) — short syntax, auto-captures outer scope
$multiplier = 3;
$numbers = [1, 2, 3, 4];
$doubled = array_map(fn($n) => $n * $multiplier, $numbers);
// [3, 6, 9, 12] — note $multiplier is auto-captured (no 'use' needed)

// Anonymous functions (closures)
$greeter = function($name) {
    return "Hi, $name!";
};
echo $greeter("Aby");  // "Hi, Aby!"

// Closures with 'use' to capture outer variables
$prefix = "Mr.";
$greetMr = function($name) use ($prefix) {
    return "$prefix $name";
};
echo $greetMr("Smith");  // "Mr. Smith"
```

## Try it yourself

Write a function `factorial(int $n): int` that calculates n! recursively. Then write the same function iteratively. Which is clearer?

## Common pitfalls

- ⚠️ Forgetting `return` makes the function return `null` silently.
- ⚠️ Default values must come AFTER non-default parameters: `function f($a = 1, $b)` is a syntax error.
- ⚠️ Using `global` to share state is a code smell — pass values in/out instead. It makes testing impossible.
