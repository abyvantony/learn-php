# Level 8: Working with files and the filesystem

**Track:** PHP · **Level:** 8/12 · **Difficulty:** `intermediate`

📚 **Today's lesson** — published 2026-06-30

## TL;DR

PHP can read, write, and manage files on the server. Common uses: log files, config files, CSV import/export, file uploads, generated PDFs.

## Real-world analogies

- Reading a file is like photocopying a document — you get a copy, the original is untouched. Writing is like replacing the page with new text.
- File handles (the resource from `fopen`) are like your hand on a book — you have to grab the book to read it, and you can only hold one at a time per hand.

## Key concepts

### `File modes`

`r` (read), `w` (write, truncates), `a` (append), `x` (write, fails if exists). The mode determines what you can do.

### `Locking`

`flock($handle, LOCK_EX)` prevents other processes from writing while you read. Critical for log files and concurrent writes.

### `Permissions`

PHP runs as a specific user (often `www-data`). It can only read files that user can access. `chmod 644` for files, `755` for directories.

### `Streams`

PHP abstracts 'reading data' so the same functions work for files, network, in-memory buffers. `php://stdin`, `php://memory`, `https://...`.

```php
<?php
// Reading an entire file
$contents = file_get_contents("/path/to/file.txt");

// Reading line by line
$lines = file("/path/to/file.txt", FILE_IGNORE_NEW_LINES);

// Writing
file_put_contents("/path/to/file.txt", "Hello, world!\n");

// Appending
file_put_contents("/path/to/log.txt", "New entry\n", FILE_APPEND);

// Reading with fopen for fine control
$handle = fopen("/path/to/file.txt", "r");
if ($handle) {
    while (($line = fgets($handle)) !== false) {
        echo $line;
    }
    fclose($handle);
}

// Checking file existence and info
file_exists("/path/to/file.txt");   // true/false
is_file("/path/to/file.txt");       // true (vs directory)
is_readable("/path/to/file.txt");   // true if PHP can read it
filesize("/path/to/file.txt");      // bytes
filemtime("/path/to/file.txt");     // last modified timestamp

// Directory operations
$entries = scandir("/path/to/dir");  // array of names, including . and ..
$entries = array_diff(scandir("/path"), ['.', '..']);  // clean version

// glob — pattern matching
$phpFiles = glob("/path/*.php");

// Recursive directory traversal
function listFiles(string $dir): array {
    $files = [];
    foreach (scandir($dir) as $entry) {
        if ($entry === '.' || $entry === '..') continue;
        $path = "$dir/$entry";
        if (is_dir($path)) {
            $files = array_merge($files, listFiles($path));
        } else {
            $files[] = $path;
        }
    }
    return $files;
}

// CSV reading
$handle = fopen("data.csv", "r");
while (($row = fgetcsv($handle)) !== false) {
    // $row is an array of fields from one line
    print_r($row);
}
fclose($handle);

// JSON reading and writing
$config = json_decode(file_get_contents("config.json"), true);  // true = assoc array
file_put_contents("config.json", json_encode($config, JSON_PRETTY_PRINT));

// File uploads (basics)
if ($_FILES['upload']['error'] === UPLOAD_ERR_OK) {
    $tmpPath = $_FILES['upload']['tmp_name'];
    $name = basename($_FILES['upload']['name']);
    move_uploaded_file($tmpPath, "/uploads/$name");
}
```

Write a script that counts how many lines, words, and characters are in a text file. Then write one that lists all `.php` files in a directory and shows their sizes in KB.

## Common pitfalls

- ⚠️ `file_get_contents` reads the whole file into memory. For huge files, use `fopen` + `fgets` in a loop.
- ⚠️ Never trust user input for file paths — use `basename()` to strip directory components. `../../etc/passwd` is a classic attack.
- ⚠️ File modes: `w` TRUNCATES the file. If you wanted to add, use `a`. There's no undo.
