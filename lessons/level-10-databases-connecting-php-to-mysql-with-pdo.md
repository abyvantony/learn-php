# Level 10: Databases: connecting PHP to MySQL with PDO

**Track:** PHP · **Level:** 10/12 · **Difficulty:** `advanced`

📚 **Today's lesson** — published 2026-07-01

## TL;DR

PHP talks to databases through extensions. PDO is the modern, database-agnostic one. You write SQL once, switch backends (MySQL, PostgreSQL, SQLite) with one config change.

## Real-world analogies

- PDO is a universal remote control. The TV (database) might be different brands, but the buttons (PDO methods) are the same.
- Prepared statements are like filling out a form with blanks — you write the structure once, then fill in the values. No one can sneak in a fake form.

## Key concepts

### `Connection string (DSN)`

A string that tells PDO which driver, host, database, and charset. `mysql:host=localhost;dbname=test;charset=utf8mb4`.

### `Prepared statements`

You write the SQL with placeholders (`?` or named like `:name`), then bind values. This prevents SQL injection — the #1 web vulnerability.

### `Fetch modes`

`PDO::FETCH_ASSOC` (assoc array), `FETCH_OBJ` (object), `FETCH_CLASS` (map to a class), `FETCH_COLUMN` (single value).

### `Transactions`

`beginTransaction()`, `commit()`, `rollBack()`. A group of queries that either all succeed or all fail. Critical for money movement.

## Code with comments

Every line has a comment. Read it slowly.

```php
<?php
// Connecting
$dsn = "mysql:host=localhost;dbname=app;charset=utf8mb4";
$pdo = new PDO($dsn, "username", "password", [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,  // throw on errors
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,         // assoc arrays by default
    PDO::ATTR_EMULATE_PREPARES   => false,                    // use real prepared statements
]);

// Querying with prepared statements (THE safe way)
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = :email");
$stmt->execute(["email" => "aby@example.com"]);
$user = $stmt->fetch();  // One row, or false if no match
$allUsers = $stmt->fetchAll();  // Array of all rows

// Insert
$stmt = $pdo->prepare("INSERT INTO users (name, email, age) VALUES (?, ?, ?)");
$stmt->execute(["Aby", "aby@example.com", 30]);
$newId = $pdo->lastInsertId();

// Update
$stmt = $pdo->prepare("UPDATE users SET name = :name WHERE id = :id");
$stmt->execute(["name" => "Aby A.", "id" => 1]);
$rowsAffected = $stmt->rowCount();

// Delete
$stmt = $pdo->prepare("DELETE FROM users WHERE id = ?");
$stmt->execute([42]);

// Preventing SQL injection — the cardinal rule
// BAD: string concatenation = SQL injection waiting to happen
$email = $_GET['email'];  // Could be: '; DROP TABLE users; --
$pdo->query("SELECT * FROM users WHERE email = '$email'");  // NEVER DO THIS

// GOOD: prepared statement
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = ?");
$stmt->execute([$email]);  // Safe no matter what $email contains

// Transactions — all or nothing
try {
    $pdo->beginTransaction();

    $pdo->prepare("UPDATE accounts SET balance = balance - ? WHERE id = ?")
        ->execute([100, 1]);

    $pdo->prepare("UPDATE accounts SET balance = balance + ? WHERE id = ?")
        ->execute([100, 2]);

    $pdo->commit();  // Both succeeded — save
} catch (Exception $e) {
    $pdo->rollBack();  // Either failed — undo everything
    throw $e;
}

// Mapping to objects
class User {
    public function __construct(
        public int $id,
        public string $name,
        public string $email,
    ) {}
}

$stmt = $pdo->prepare("SELECT id, name, email FROM users");
$stmt->setFetchMode(PDO::FETCH_CLASS, User::class);
$stmt->execute();
$users = $stmt->fetchAll();  // Array of User objects
```

## Try it yourself

Build a `Notes` class that wraps PDO. Methods: `create($title, $body)`, `find($id)`, `update($id, $title, $body)`, `delete($id)`, `all()`. All use prepared statements.

## Common pitfalls

- ⚠️ NEVER concatenate user input into SQL. Use prepared statements. Always. No exceptions.
- ⚠️ Default PDO error mode is silent (returns false). Always set `ERRMODE_EXCEPTION` so you see what went wrong.
- ⚠️ `fetch()` returns `false` if no row found. `fetchAll()` returns empty array. Test for both, not the same thing.
