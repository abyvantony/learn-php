# Level 11: Composer, autoloading, and the modern PHP ecosystem

**Track:** PHP · **Level:** 11/12 · **Difficulty:** `advanced`

📚 **Today's lesson** — published 2026-07-01

## TL;DR

Composer is PHP's package manager. Autoloading means you don't have to `require` every file — classes load automatically when you first use them. Together they make modern PHP possible.

## Real-world analogies

- Composer is PHP's App Store. Need a routing library, HTTP client, or testing framework? Someone's already built it. `composer require` it.
- Autoloading is like a librarian who, when you ask for a book, knows exactly which shelf it's on and brings it to you. You don't have to hunt through the stacks.

## Key concepts

### ``composer.json``

Manifest that lists your dependencies, autoloading rules, and project metadata. Run `composer require vendor/package` to add a dep.

### `PSR-4 autoloading`

Convention: one class per file, namespace matches folder structure. `App\Models\User` lives in `src/Models/User.php`.

### ``vendor/``

Where Composer puts downloaded packages. Always gitignored — never commit it.

### ``composer.lock``

Records the EXACT versions installed. Commit it for apps (everyone gets the same versions). Don't for libraries (consumers get their own).

```php
    "name": "aby/my-app",
    "description": "Learning PHP with Composer",
    "type": "project",
    "require": {
        "php": ">=8.1",
        "guzzlehttp/guzzle": "^7.0",        // HTTP client
        "monolog/monolog": "^3.0",         // logging
        "vlucas/phpdotenv": "^5.0"         // .env file loader
    },
    "require-dev": {
        "phpunit/phpunit": "^10.0",        // testing
        "phpstan/phpstan": "^1.10"         // static analysis
    },
    "autoload": {
        "psr-4": {
            "App\\": "src/"              // App\Models\User → src/Models/User.php
        }
    },
    "autoload-dev": {
        "psr-4": {
            "App\\Tests\\": "tests/"
        }
    }
}
```

```php
<?php
// index.php — the only file you need to require manually
require __DIR__ . '/vendor/autoload.php';

use App\Models\User;
use GuzzleHttp\Client;
use Monolog\Logger;
use Monolog\Handler\StreamHandler;

// Composer autoloaded everything. No more `require 'src/Models/User.php'`!

$log = new Logger('app');
$log->pushHandler(new StreamHandler('app.log', Logger::WARNING));

$log->warning('Something happened');
$log->error('Something bad happened');

$client = new Client();
$response = $client->get('https://api.github.com/users/octocat');
$data = json_decode($response->getBody(), true);

$user = new User('Aby', 'aby@example.com');
```

```bash
# Common Composer commands
composer init                  # Create composer.json interactively
composer require vendor/pkg     # Add a dependency
composer install                # Install from composer.lock
composer update                 # Upgrade to latest versions
composer dump-autoload          # Regenerate the autoloader
composer outdated               # See which deps have updates
```

```php
<?php
// Manual PSR-4 autoloader (if you don't want Composer)
spl_autoload_register(function (string $class) {
    $prefix = 'App\\';
    $baseDir = __DIR__ . '/src/';
    if (str_starts_with($class, $prefix)) {
        $relative = substr($class, strlen($prefix));
        $file = $baseDir . str_replace('\\', '/', $relative) . '.php';
        if (file_exists($file)) {
            require $file;
        }
    }
});
```

## Common pitfalls

- ⚠️ Don't commit `vendor/` — it's huge and not portable. Use `.gitignore`.
- ⚠️ Composer requires PHP installed and on your PATH. If `composer` command not found, install from getcomposer.org.
- ⚠️ Updating a single package: `composer update vendor/package`, not `composer update` (which updates everything and can break things).
